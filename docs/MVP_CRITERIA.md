# MVP Criteria

## Objetivo

Definir que significa que el MVP de `PuntoVenta` esta listo para validacion.

## Criterios funcionales

- [ ] El flujo principal del usuario esta definido.
- [ ] El flujo principal funciona de punta a punta.
- [ ] Los errores esperados tienen mensajes claros.
- [ ] Los datos importantes se guardan de forma persistente si el MVP es piloto real.
- [ ] No hay P0/P1 abiertos.

## Criterios tecnicos

- [ ] Hay repo Git local.
- [ ] Hay docs base.
- [ ] Hay tareas y handoffs.
- [ ] Hay forma repetible de ejecutar/verificar localmente.
- [ ] Hay estrategia de deploy si aplica.
- [ ] No hay secretos en repo.

## Criterios QA

- [ ] QA tiene checklist.
- [ ] QA valido happy path.
- [ ] QA valido casos negativos principales.
- [ ] QA documento riesgos o pendientes.

## Fuera de alcance inicial

Documentar aqui lo que no entra en MVP para evitar expansion accidental.
