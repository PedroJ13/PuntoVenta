# QA Test Plan

## Objetivo

Validar el MVP de `PuntoVenta` sin aprobar tareas con P0/P1 abiertos.

## Severidad

- P0: bloquea todo el sistema o causa perdida/exposicion grave de datos.
- P1: bloquea flujo principal o regla critica.
- P2: problema importante con workaround.
- P3: detalle menor, visual o copy.

## Checklist inicial

- Flujo de caja: `docs/QA_CHECKLIST_FLUJO_CAJA.md`.
- [ ] Abrir la app o flujo principal.
- [ ] Validar happy path.
- [ ] Validar campos requeridos.
- [ ] Validar errores esperados.
- [ ] Validar responsive si hay frontend.
- [ ] Validar permisos/aislamiento si aplica.
- [ ] Validar persistencia si aplica.

## Formato de hallazgo

```text
Flujo:
Paso:
Resultado esperado:
Resultado observado:
Severidad sugerida:
Evidencia:
```
